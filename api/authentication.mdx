# Authentication

All Salsa as a Service API requests require authentication to ensure secure access to your data. We support two authentication methods: API Key authentication (recommended for most use cases) and OAuth 2.0 (for third-party integrations).

## API Key Authentication

API Key authentication is the simplest and most common method for authenticating with our API. This method is ideal for server-to-server integrations and internal tools.

### Getting Your API Key

1. Log in to your Salsa as a Service account
2. Navigate to [Developer Settings](https://app.salsaasaservice.com/settings/api)
3. Click "Generate New API Key"
4. Copy and securely store your API key

<Warning>
  **Keep Your API Key Secret**: Never expose your API key in client-side code, public repositories, or version control. Treat it like a password.
</Warning>

### Using Your API Key

Include your API key in the `Authorization` header of every request using the Bearer token format:

```http
Authorization: Bearer YOUR_API_KEY
```

<CodeGroup>

```go Go
package main

import (
    "fmt"

    "net/http"
)

func main() {
    client := &http.Client{}

    body, _ := ioutil.ReadAll(resp.Body)

}
```

### API Key Types

We provide two types of API keys for different environments:

<Tabs>
  <Tab title="Production Keys">
    **Format**: `prod_live_...`
    
    **Usage**: Use production keys for live environments and real data
    
    **Access**: Full access to all production resources and data
    
    **Rate Limits**: Based on your subscription plan
  </Tab>
  <Tab title="Sandbox Keys">
    **Format**: `sandbox_test_...`
    
    **Usage**: Use sandbox keys for testing and development
    
    **Access**: Access to sandbox environment with test data only
    
    **Rate Limits**: Same as production, but requests don't count toward production limits
  </Tab>
</Tabs>

### Managing API Keys

You can manage your API keys from the Developer Settings page:

- **Create**: Generate new API keys for different applications
- **Rotate**: Regularly rotate keys for security best practices
- **Revoke**: Immediately revoke compromised keys
- **Monitor**: View usage statistics for each key

<Note>
  **Key Rotation**: We recommend rotating your API keys every 90 days for enhanced security. You can have multiple active keys to facilitate zero-downtime rotation.
</Note>

## OAuth 2.0 Authentication

OAuth 2.0 is ideal for third-party applications that need to access user data without exposing API keys. This method allows users to grant limited access to their Salsa as a Service account.

### OAuth 2.0 Flow

We support the Authorization Code flow with PKCE (Proof Key for Code Exchange):

1. **Authorization Request**: Redirect user to authorization URL
2. **User Consent**: User approves requested permissions
3. **Authorization Code**: Redirect back to your app with code
4. **Token Exchange**: Exchange code for access token
5. **API Requests**: Use access token to make authenticated requests

### Registering Your Application

1. Go to [Developer Settings](https://app.salsaasaservice.com/settings/api)
2. Click "Create OAuth Application"
3. Provide application details:
   - Application name
   - Redirect URIs
   - Application description
   - Privacy policy URL
4. Receive your Client ID and Client Secret

### OAuth Endpoints

```
Authorization URL: https://app.salsaasaservice.com/oauth/authorize
Token URL: https://api.salsaasaservice.com/oauth/token
Revocation URL: https://api.salsaasaservice.com/oauth/revoke
```

### Authorization Request

Redirect users to the authorization URL with the following parameters:

```http
GET https://app.salsaasaservice.com/oauth/authorize?
  response_type=code&
  client_id=YOUR_CLIENT_ID&
  redirect_uri=https://your-app.com/callback&
  scope=read:ingredients write:recipes read:analytics&
  state=random_state_string
```

### OAuth 2.0 Implementation Examples

<CodeGroup>
```javascript JavaScript
// Step 1: Redirect to authorization URL
const authUrl = new URL('https://app.salsaasaservice.com/oauth/authorize');
authUrl.searchParams.set('response_type', 'code');
authUrl.searchParams.set('client_id', 'YOUR_CLIENT_ID');
authUrl.searchParams.set('redirect_uri', 'https://your-app.com/callback');
authUrl.searchParams.set('scope', 'read:ingredients write:recipes');
authUrl.searchParams.set('state', generateRandomState());

window.location.href = authUrl.toString();

// Step 2: Exchange code for token (in your callback handler)
const tokenResponse = await fetch('https://api.salsaasaservice.com/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    grant_type: 'authorization_code',
    code: authorizationCode,
    client_id: 'YOUR_CLIENT_ID',
    client_secret: 'YOUR_CLIENT_SECRET',
    redirect_uri: 'https://your-app.com/callback'
  })
});

const { access_token, refresh_token } = await tokenResponse.json();

// Step 3: Use access token for API requests
const apiResponse = await fetch('https://api.salsaasaservice.com/v2/ingredients', {
  headers: {
    'Authorization': `Bearer ${access_token}`
  }
});
```

```python Python
token_response = requests.post(
```

### OAuth Scopes

Request only the scopes your application needs:

| Scope | Description |
|-------|-------------|
| `read:ingredients` | Read ingredient inventory data |
| `write:ingredients` | Create and update ingredients |
| `read:recipes` | Read recipe information |
| `write:recipes` | Create and update recipes |
| `read:production` | Read production batch data |
| `write:production` | Create and update production batches |
| `read:analytics` | Access analytics and reports |
| `read:webhooks` | Read webhook configurations |
| `write:webhooks` | Create and update webhooks |

### Refreshing Access Tokens

Access tokens expire after 1 hour. Use the refresh token to obtain a new access token:

<CodeGroup>
```javascript JavaScript
const refreshResponse = await fetch('https://api.salsaasaservice.com/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    grant_type: 'refresh_token',
    refresh_token: 'YOUR_REFRESH_TOKEN',
    client_id: 'YOUR_CLIENT_ID',
    client_secret: 'YOUR_CLIENT_SECRET'
  })
});

const { access_token, refresh_token } = await refreshResponse.json();
```

```python Python
refresh_response = requests.post(
    'https://api.salsaasaservice.com/oauth/token',
    json={
        'grant_type': 'refresh_token',
        'refresh_token': refresh_token,
        'client_id': 'YOUR_CLIENT_ID',
        'client_secret': 'YOUR_CLIENT_SECRET'
    }
)

new_tokens = refresh_response.json()
access_token = new_tokens['access_token']
```

```php PHP
<?php
$refreshData = [
    'grant_type' => 'refresh_token',
```

## Security Best Practices

Follow these security guidelines to protect your API credentials and user data:

### API Key Security

<CardGroup cols={2}>
  <Card title="Environment Variables" icon="shield-check">
    Store API keys in environment variables, never in source code:
    ```bash
    export SALSA_API_KEY="your_api_key"
    ```
  </Card>
  <Card title="Key Rotation" icon="rotate">
    Rotate API keys regularly (recommended every 90 days) and immediately if compromised.
  </Card>
  <Card title="Least Privilege" icon="lock">
    Create separate API keys for different applications with minimal required permissions.
  </Card>
  <Card title="Monitor Usage" icon="chart-line">
    Regularly review API key usage logs in Developer Settings for suspicious activity.
  </Card>
</CardGroup>

### OAuth Security

<Warning>
  **State Parameter**: Always use a random, unique `state` parameter to prevent CSRF attacks. Verify the state matches when receiving the callback.
</Warning>

<Note>
  **HTTPS Only**: All OAuth redirect URIs must use HTTPS in production. HTTP is only allowed for localhost during development.
</Note>

### Additional Headers

While not required, these headers improve security and debugging:

```http
Content-Type: application/json
X-Request-ID: unique-request-identifier
User-Agent: YourApp/1.0
```

## Authentication Errors

Common authentication error responses:

### 401 Unauthorized

```json
{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid or missing API key",
    "details": {
      "reason": "The provided API key is invalid or has been revoked"
    }
  }
}
```

**Common causes:**
- Missing `Authorization` header
- Invalid API key format
- Revoked or expired API key
- Using sandbox key for production endpoint

### 403 Forbidden

```json
{
  "error": {
    "code": "FORBIDDEN",
    "message": "Insufficient permissions",
    "details": {
      "required_scope": "write:recipes",
      "current_scopes": ["read:recipes"]
    }
  }
}
```

**Common causes:**
- API key lacks required permissions
- OAuth token missing required scopes
- Resource access restricted to account owner
