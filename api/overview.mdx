# API Overview

Welcome to the Salsa as a Service API v2.0. Our REST API enables you to manage inventory, track production, create recipes, and analyze your salsa operations programmatically.

## Base URL

All API requests should be made to:

```
https://api.salsaasaservice.com/v2
```

<Note>
  **New in v2.0**: GraphQL endpoint available at `https://api.salsaasaservice.com/graphql`
</Note>

## Rate Limits

API rate limits are enforced based on your subscription plan:

| Plan | Requests Per Hour | Burst Limit |
|------|------------------|-------------|
| Starter | 1,000 | 100/min |
| Professional | 10,000 | 500/min |
| Enterprise | 100,000 | 2,000/min |

### Rate Limit Headers

Every API response includes rate limit information in the headers:

```http
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1640995200
```

<CodeGroup>
```javascript JavaScript
// Check rate limit status
const response = await fetch('https://api.salsaasaservice.com/v2/ingredients', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const limit = response.headers.get('X-RateLimit-Limit');
const remaining = response.headers.get('X-RateLimit-Remaining');
const reset = response.headers.get('X-RateLimit-Reset');

console.log(`${remaining}/${limit} requests remaining`);
```

```python Python
import requests
from datetime import datetime

response = requests.get(
    'https://api.salsaasaservice.com/v2/ingredients',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

limit = response.headers.get('X-RateLimit-Limit')
remaining = response.headers.get('X-RateLimit-Remaining')
reset_timestamp = int(response.headers.get('X-RateLimit-Reset'))
reset_time = datetime.fromtimestamp(reset_timestamp)

print(f"{remaining}/{limit} requests remaining")
print(f"Rate limit resets at: {reset_time}")
```

```php PHP
<?php
$ch = curl_init('https://api.salsaasaservice.com/v2/ingredients');
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Authorization: Bearer YOUR_API_KEY'
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);
curl_setopt($ch, CURLOPT_HEADER, true);

$response = curl_exec($ch);
$header_size = curl_getinfo($ch, CURLINFO_HEADER_SIZE);
$headers = substr($response, 0, $header_size);

preg_match('/X-RateLimit-Limit: (\d+)/', $headers, $limit);
preg_match('/X-RateLimit-Remaining: (\d+)/', $headers, $remaining);

echo "{$remaining[1]}/{$limit[1]} requests remaining\n";
?>
```
</CodeGroup>

<Warning>
  **Rate Limit Exceeded**: When you exceed your rate limit, the API returns a `429 Too Many Requests` status code. Implement exponential backoff in your applications to handle rate limiting gracefully.
</Warning>

## Webhooks

Configure webhooks to receive real-time notifications about critical events in your salsa operations.

### Supported Events

| Event | Description | Payload |
|-------|-------------|----------|
| `ingredient.low_stock` | Triggered when ingredient stock falls below threshold | Ingredient ID, current stock, threshold |
| `production.batch_completed` | Fired when a production batch is marked complete | Batch ID, recipe ID, quality score |
| `quality.alert` | Sent when quality scores fall below acceptable levels | Batch ID, quality score, threshold |
| `inventory.expiry_warning` | Alert for ingredients approaching expiry date | Ingredient ID, expiry date, days remaining |

### Setting Up Webhooks

<CodeGroup>
```javascript JavaScript
const response = await fetch('https://api.salsaasaservice.com/v2/webhooks', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    url: 'https://your-app.com/webhooks/salsa',
    events: ['ingredient.low_stock', 'production.batch_completed'],
    secret: 'your_webhook_secret'
  })
});

const webhook = await response.json();
```

```python Python
import requests

response = requests.post(
    'https://api.salsaasaservice.com/v2/webhooks',
    headers={'Authorization': 'Bearer YOUR_API_KEY'},
    json={
        'url': 'https://your-app.com/webhooks/salsa',
        'events': ['ingredient.low_stock', 'production.batch_completed'],
        'secret': 'your_webhook_secret'
    }
)

webhook = response.json()
```

```php PHP
<?php
$data = [
    'url' => 'https://your-app.com/webhooks/salsa',
    'events' => ['ingredient.low_stock', 'production.batch_completed'],
    'secret' => 'your_webhook_secret'
];

$ch = curl_init('https://api.salsaasaservice.com/v2/webhooks');
curl_setopt($ch, CURLOPT_POST, true);
curl_setopt($ch, CURLOPT_POSTFIELDS, json_encode($data));
curl_setopt($ch, CURLOPT_HTTPHEADER, [
    'Authorization: Bearer YOUR_API_KEY',
    'Content-Type: application/json'
]);
curl_setopt($ch, CURLOPT_RETURNTRANSFER, true);

$response = curl_exec($ch);
$webhook = json_decode($response);
?>
```
</CodeGroup>

<Note>
  **Webhook Security**: Always use the `secret` parameter to verify webhook authenticity. Each webhook request includes an `X-Salsa-Signature` header for verification.
</Note>

## Error Handling

The API uses standard HTTP status codes and returns detailed error information:

### Error Response Format

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing required fields",
    "details": {
      "missing_fields": ["name", "category"]
    }
  }
}
```

### HTTP Status Codes

| Code | Status | Description |
|------|--------|-------------|
| 200 | OK | Request succeeded |
| 201 | Created | Resource successfully created |
| 400 | Bad Request | Invalid request data |
| 401 | Unauthorized | Missing or invalid API key |
| 403 | Forbidden | Insufficient permissions |
| 404 | Not Found | Resource not found |
| 422 | Unprocessable Entity | Validation errors |
| 429 | Too Many Requests | Rate limit exceeded |
| 500 | Internal Server Error | Server error |
| 502 | Bad Gateway | Upstream service error |
| 503 | Service Unavailable | Service temporarily unavailable |
