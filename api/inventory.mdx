# Inventory Management API

The Inventory Management API enables you to track ingredients, manage stock levels, monitor expiry dates, and maintain quality scores for all ingredients used in salsa production.

## Base URL

```
https://api.salsaasaservice.com/v2/ingredients
```

---

## Get All Ingredients

Retrieve a paginated list of all ingredients in your inventory with detailed stock information.

<CodeGroup>
```http Request
GET /ingredients
Authorization: Bearer YOUR_API_KEY
```

```bash cURL
curl -X GET "https://api.salsaasaservice.com/v2/ingredients" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch('https://api.salsaasaservice.com/v2/ingredients', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.salsaasaservice.com/v2/ingredients',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

data = response.json()
```
</CodeGroup>

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="per_page" type="integer" default="50">
  Number of items per page (max: 100)
</ParamField>

<ParamField query="category" type="string">
  Filter by ingredient category (vegetables, spices, herbs, dairy, etc.)
</ParamField>

<ParamField query="supplier" type="string">
  Filter by supplier name
</ParamField>

<ParamField query="low_stock" type="boolean">
  Show only ingredients below minimum stock threshold
</ParamField>

### Response

<ResponseField name="data" type="array">
  Array of ingredient objects
  
  <Expandable title="Ingredient Object">
    <ResponseField name="id" type="string">
      Unique ingredient identifier (e.g., `ing_123`)
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Ingredient name
    </ResponseField>
    
    <ResponseField name="category" type="string">
      Ingredient category (vegetables, spices, herbs, dairy, etc.)
    </ResponseField>
    
    <ResponseField name="current_stock" type="number">
      Current stock quantity
    </ResponseField>
    
    <ResponseField name="unit" type="string">
      Unit of measurement (lbs, oz, kg, count, etc.)
    </ResponseField>
    
    <ResponseField name="supplier" type="string">
      Supplier or vendor name
    </ResponseField>
    
    <ResponseField name="last_updated" type="string">
      ISO 8601 timestamp of last stock update
    </ResponseField>
    
    <ResponseField name="expiry_date" type="string">
      ISO 8601 timestamp when ingredient expires
    </ResponseField>
    
    <ResponseField name="quality_score" type="number">
      Quality rating from 0-10 based on freshness and condition
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination metadata
  
  <Expandable title="Pagination Object">
    <ResponseField name="page" type="integer">
      Current page number
    </ResponseField>
    
    <ResponseField name="per_page" type="integer">
      Items per page
    </ResponseField>
    
    <ResponseField name="total" type="integer">
      Total number of ingredients
    </ResponseField>
  </Expandable>
</ResponseField>

## Add New Ingredient

Add a new ingredient to your inventory system.

<CodeGroup>
```http Request
POST /ingredients
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
```

```bash cURL
curl -X POST "https://api.salsaasaservice.com/v2/ingredients" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Jalapeño Peppers",
    "category": "vegetables",
    "initial_stock": 5.0,
    "unit": "lbs",
    "supplier": "Spice Garden",
    "expiry_date": "2024-01-25T00:00:00Z"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.salsaasaservice.com/v2/ingredients', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Jalapeño Peppers',
    category: 'vegetables',
    initial_stock: 5.0,
    unit: 'lbs',
    supplier: 'Spice Garden',
    expiry_date: '2024-01-25T00:00:00Z'
  })
});
```
</CodeGroup>

### Request Body

<ParamField body="name" type="string" required>
  Ingredient name
</ParamField>

<ParamField body="category" type="string" required>
  Category: `vegetables`, `spices`, `herbs`, `dairy`, `meats`, `condiments`, `other`
</ParamField>

<ParamField body="initial_stock" type="number" required>
  Initial stock quantity
</ParamField>

<ParamField body="unit" type="string" required>
  Unit of measurement: `lbs`, `oz`, `kg`, `g`, `count`, `gallons`, `liters`
</ParamField>

<ParamField body="supplier" type="string">
  Supplier or vendor name
</ParamField>

<ParamField body="expiry_date" type="string">
  Expiry date in ISO 8601 format
</ParamField>

<ParamField body="min_stock_threshold" type="number">
  Minimum stock level that triggers low stock alerts
</ParamField>

### Response

Returns the created ingredient object with generated ID and metadata.

```json
{
  "id": "ing_789",
  "name": "Jalapeño Peppers",
  "category": "vegetables",
  "current_stock": 5.0,
  "unit": "lbs",
  "supplier": "Spice Garden",
  "expiry_date": "2024-01-25T00:00:00Z",
  "quality_score": 10.0,
  "created_at": "2024-01-15T12:00:00Z",
  "last_updated": "2024-01-15T12:00:00Z"
}
```

## Update Ingredient Stock

Update the stock level for a specific ingredient. This endpoint is used when receiving new shipments or adjusting inventory counts.

<CodeGroup>
```http Request
PUT /ingredients/{ingredient_id}/stock
Authorization: Bearer YOUR_API_KEY
Content-Type: application/json
```

```bash cURL
curl -X PUT "https://api.salsaasaservice.com/v2/ingredients/ing_123/stock" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "quantity": 30.0,
    "unit": "lbs",
    "notes": "New shipment received"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.salsaasaservice.com/v2/ingredients/ing_123/stock', {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    quantity: 30.0,
    unit: 'lbs',
    notes: 'New shipment received'
  })
});
```
</CodeGroup>

### Path Parameters

<ParamField path="ingredient_id" type="string" required>
  The unique identifier of the ingredient
</ParamField>

### Request Body

<ParamField body="quantity" type="number" required>
  New stock quantity (replaces current stock)
</ParamField>

<ParamField body="unit" type="string" required>
  Unit of measurement (must match ingredient's unit)
</ParamField>

<ParamField body="notes" type="string">
  Optional notes about the stock update
</ParamField>

<ParamField body="operation" type="string" default="set">
  Stock operation: `set` (replace), `add` (increase), or `subtract` (decrease)
</ParamField>

### Response

```json
{
  "id": "ing_123",
  "name": "Roma Tomatoes",
  "current_stock": 30.0,
  "unit": "lbs",
  "previous_stock": 25.5,
  "updated_at": "2024-01-15T11:00:00Z"
}
```

Retrieve detailed information about a specific ingredient.

```http

Authorization: Bearer YOUR_API_KEY
```

### Path Parameters

<ParamField path="ingredient_id" type="string" required>
  The unique identifier of the ingredient
</ParamField>

### Response

Returns a single ingredient object with complete details.

## Delete Ingredient

Remove an ingredient from your inventory. This operation cannot be undone.

```http
DELETE /ingredients/{ingredient_id}
Authorization: Bearer YOUR_API_KEY
```

<Warning>
  Deleting an ingredient will not remove it from existing recipes, but those recipes will show a warning about missing ingredients.
</Warning>

### Path Parameters

<ParamField path="ingredient_id" type="string" required>
  The unique identifier of the ingredient to delete
</ParamField>

### Response

```json
{
  "success": true,
  "message": "Ingredient successfully deleted",
  "deleted_id": "ing_123"
}
```

## Error Responses

The Inventory API returns standard HTTP status codes and detailed error messages.

### 400 Bad Request

```json
{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing required fields",
    "details": {
      "missing_fields": ["name", "category"]
    }
  }
}
```

### 404 Not Found

```json
{
  "error": {
    "code": "INGREDIENT_NOT_FOUND",
    "message": "Ingredient with ID 'ing_123' does not exist"
  }
}
```

### 422 Unprocessable Entity

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid stock quantity",
    "details": {
      "quantity": "Must be a positive number"
    }
  }
}
```
