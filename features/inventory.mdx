## Getting Started

<Steps>
  <Step title="Connect IoT Sensors">
    Install RFID tags and temperature sensors in your storage areas. Our hardware partners provide easy-to-install sensor kits that integrate seamlessly with the platform.
    
    <Info>
      IoT sensors automatically sync with the cloud every 30 seconds, providing real-time visibility into your inventory.
    </Info>
  </Step>
  <Step title="Enable AI Features">
    Activate AI predictions and import 90 days of historical data. The AI engine requires at least 60 days of data to generate accurate predictions.
    
    ```javascript
    // Example: Enable AI predictions
    const aiConfig = await salsa.ai.inventory.enable({
      location: "kitchen_1",
      historical_data_days: 90,
      prediction_horizon: 14
    });
    ```
  </Step>
  <Step title="Configure Auto-Ordering">
    Set up automated ordering rules and supplier preferences. Define approval workflows for orders above certain thresholds.
  </Step>
  <Step title="Monitor & Optimize">
    Let AI learn from your patterns and optimize over time. Review weekly insights to understand how the system is improving your operations.
  </Step>
</Steps>

```javascript

```

## IoT Dashboard

Real-time monitoring of all your connected sensors provides unprecedented visibility into your inventory.

### Live Sensor Monitoring

<Tabs>
  <Tab title="Temperature Monitoring">
    Monitor temperature across all storage areas:
    
    - **Walk-in Coolers**: Ensure 34-38°F range
    - **Freezers**: Maintain 0°F or below
    - **Dry Storage**: Track ambient temperature
    - **Prep Areas**: Monitor food safety zones
    
    <Info>
      Automatic alerts trigger when temperatures deviate from safe ranges for more than 5 minutes.
    </Info>
    
    ```javascript
    // Example: Query temperature sensors
    const temps = await salsa.iot.sensors.temperature.getAll({
      location: "kitchen_1",
      status: "active"
    });
    
    temps.forEach(sensor => {
      console.log(`${sensor.name}: ${sensor.current_temp}°F`);
      if (sensor.is_out_of_range) {
        console.log(`⚠️  Alert: ${sensor.alert_message}`);
      }
    });
    ```
  </Tab>
  
  <Tab title="RFID Tracking">
    Track ingredient movement with RFID technology:
    
    - **Automatic check-in** when deliveries arrive
    - **Location tracking** within your facility
    - **First-in-first-out (FIFO)** enforcement
    - **Expiry date tracking** with automatic alerts
    
    RFID tags provide 99.9% accuracy compared to manual tracking.
    
    ```javascript
    // Example: Track ingredient with RFID
    const rfidReading = await salsa.iot.rfid.scan({
      tag_id: "RFID-TOM-001",
      action: "move",
      from_location: "storage_cooler_1",
      to_location: "prep_station_2"
    });
    
    console.log(`Tracked: ${rfidReading.ingredient_name}`);
    console.log(`Days until expiry: ${rfidReading.days_to_expiry}`);
    ```
  </Tab>
  
  <Tab title="Weight Sensors">
    Continuously monitor ingredient quantities:
    
    - **Real-time weight readings** updated every 30 seconds
    - **Automatic stock updates** when ingredients are used
    - **Theft detection** for unusual weight changes
    - **Yield tracking** for better cost analysis
    
    <Tip>
      Place weight sensors under your most expensive ingredients to track usage and prevent waste.
    </Tip>
  </Tab>
  
  <Tab title="Freshness Detection">
    Advanced sensors detect ingredient freshness:
    
    - **Gas sensors** detect ethylene and spoilage compounds
    - **Color analysis** identifies visual degradation
    - **Texture sensors** for produce firmness
    - **pH monitoring** for acidic ingredients
    
    AI combines all sensor data to predict optimal usage dates, often more accurate than printed expiration dates.
  </Tab>
</Tabs>

## API Reference

### Core Inventory Endpoints

#### Get All Ingredients

```http
GET /v2/ingredients
```

Retrieve a list of all ingredients in your inventory with current stock levels, IoT sensor data, and AI predictions.

**Query Parameters:**
- `location` (string): Filter by location ID
- `category` (string): Filter by ingredient category
- `low_stock` (boolean): Only show low stock items
- `include_predictions` (boolean): Include AI predictions in response

**Response:**
```json
{
  "data": [
    {
      "id": "ing_123",
      "name": "Roma Tomatoes",
      "category": "vegetables",
      "current_stock": 25.5,
      "unit": "lbs",
      "supplier": "Fresh Farms Co.",
      "last_updated": "2024-01-15T10:30:00Z",
      "expiry_date": "2024-01-20T00:00:00Z",
      "quality_score": 8.5,
      "iot_data": {
        "temperature": 36.5,
        "rfid_last_scan": "2024-01-15T10:25:00Z",
        "weight_sensor_reading": 25.4
      },
      "ai_predictions": {
        "predicted_depletion_date": "2024-01-18",
        "recommended_reorder_quantity": 50.0,
        "confidence": 94.5
      }
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 50,
    "total": 127
  }
}
```

#### Update Ingredient Stock

```http
PUT /v2/ingredients/{ingredient_id}/stock
```

**Request Body:**
```json
{
  "quantity": 30.0,
  "unit": "lbs",
  "transaction_type": "purchase",
  "notes": "New shipment received",
  "supplier_id": "sup_456",
  "cost_per_unit": 2.50
}
```

#### Enable AI Auto-Ordering

```http
POST /v2/ai/inventory/auto-order
```

**Request Body:**
```json
{
  "location": "kitchen_1",
  "enabled": true,
  "max_order_value": 5000,
  "approval_required_over": 1000,
  "preferred_suppliers": ["sysco", "us_foods"],
  "reorder_trigger": "forecast_based",
  "lead_time_buffer_days": 2,
  "safety_stock_multiplier": 1.5
}
```

<Steps>

## Troubleshooting

<Accordion title="IoT sensors showing offline status">
  **Common causes:**
  - WiFi connectivity issues in storage areas
  - Sensor batteries need replacement (lasts 6-12 months)
  - Sensor firmware needs updating
  
  **Solutions:**
  - Check WiFi signal strength in sensor locations
  - Replace batteries (LED indicator will flash red when low)
  - Update firmware via the mobile app
  
  <Info>
    Sensors automatically reconnect when connectivity is restored. Historical data is preserved.
  </Info>
</Accordion>

<Accordion title="AI predictions seem inaccurate">
  **Common causes:**
  - Insufficient historical data (need at least 60 days)
  - Recent business changes not reflected in data
  - Unusual events affecting normal patterns
  
  **Solutions:**
  - Wait for more data collection (predictions improve over time)
  - Mark unusual periods as "anomalies" so AI learns to ignore them
  - Adjust seasonal settings in AI configuration
  
  <Tip>
    AI prediction accuracy improves by 5-10% each month as it learns your patterns.
  </Tip>
</Accordion>

<Accordion title="Auto-ordering not triggering">
  **Checklist:**
  - [ ] Auto-ordering is enabled in settings
  - [ ] Supplier integrations are configured
  - [ ] Stock levels are being updated (manually or via IoT)
  - [ ] Reorder points are set correctly
  
  Check the auto-order logs in Settings > Inventory > Auto-Order > Logs to see detailed status.
</Accordion>
