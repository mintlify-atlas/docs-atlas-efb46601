### Key Forecast Components

<Accordion title="Confidence Intervals">
  Every prediction includes a confidence interval (default 95%) showing the range where actual demand is likely to fall. Narrower intervals indicate higher certainty.
  
  **Example**: A prediction of 12.5 gallons with interval [11.2, 13.8] means there's a 95% chance actual demand will be between 11.2 and 13.8 gallons.
</Accordion>

<Accordion title="Contributing Factors">
  The AI explains which factors influenced each prediction:
  
  - **Historical Trend**: Long-term demand changes (growth or decline)
  - **Day of Week**: Weekly patterns (e.g., Friday rush, Monday slow)
  - **Weather**: Temperature, precipitation, and seasonal conditions
  - **Local Events**: Concerts, sports games, festivals within 2 miles
  - **Seasonality**: Monthly and seasonal demand patterns
</Accordion>

<Accordion title="Actionable Recommendations">
  Each forecast includes specific actions:
  
  - **Production adjustments**: Increase/decrease batch quantities
  - **Timing recommendations**: Optimal production start times
  - **Ingredient orders**: Suggested reorder quantities
  - **Staffing suggestions**: Recommended labor allocation
</Accordion>

### Accuracy Dashboard

Access detailed accuracy metrics in your dashboard:

```python

    start_date="2024-01-01",
```

### Multi-Recipe Forecasting

Forecast demand for your entire menu simultaneously:

```javascript
// Get forecasts for all recipes
const forecasts = await salsa.ai.forecast.getMultiRecipe({
  location_id: "loc_123",
  days: 7,
  recipes: ["house_salsa", "verde_salsa", "mango_salsa", "habanero_salsa"],
  include_recommendations: true
});

// AI optimizes production schedule across all recipes

  console.log(`${recipe.name}: ${recipe.predicted_demand} ${recipe.unit}`);

});
```

## Troubleshooting

<Accordion title="Low Forecast Accuracy (under 90%)">
  **Common causes**:
  
  - Insufficient historical data (need 90+ days)
  - Missing external data sources (weather, events)
  - Inconsistent POS sync causing data gaps
  - Recent menu changes not reflected in model
  
  **Solutions**:
  
  1. Verify POS integration is syncing correctly
  2. Connect additional data sources
  3. Wait 2-4 weeks for model to adapt to changes
  4. Contact support if accuracy doesn't improve
</Accordion>

<Accordion title="Forecasts Not Updating">
  **Check**:
  
  - POS connection status in Settings > Integrations
  - Last sync timestamp in forecast dashboard
  - API key validity and permissions
  - Data source connection health
  
  **Resolution**: Navigate to Settings > Integrations and click "Reconnect" for each data source.
</Accordion>

<Accordion title="Unexpected Demand Spikes Not Predicted">
  The AI requires historical examples to predict unusual events. After 2-3 occurrences of similar events, the model will learn the pattern.
  
  **Manual Override**: You can manually adjust forecasts for known one-time events in the dashboard.
</Accordion>
