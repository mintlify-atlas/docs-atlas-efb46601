### Integration Examples

<CodeGroup>
```javascript Custom Dashboard Integration
// Build custom production dashboard
const salsa = new SalsaClient({ apiKey: 'prod_key_...' });

// Get today's production data
const production = await salsa.production.getBatches({
  date: new Date(),
  status: ['in_progress', 'completed']
});

// Get quality scores
const quality = await salsa.quality.getScores({
  batch_ids: production.map(b => b.id)
});

// Display in your custom dashboard
renderDashboard({ production, quality });
```

```python ERP Sync Integration
from salsa_sdk import SalsaClient
import erp_system

client = SalsaClient(api_key="prod_key_...")

# Get completed batches
batches = client.production.get_batches(
    status="completed",
    start_date="2024-01-01"
)

# Sync to ERP system

    erp_system.create_production_order({
```

```php Webhook Handler
<?php
require 'vendor/autoload.php';

use Salsa\Webhook;

// Verify webhook signature
$webhook = new Webhook($_ENV['WEBHOOK_SECRET']);
if (!$webhook->verify($_SERVER['HTTP_X_SALSA_SIGNATURE'], file_get_contents('php://input'))) {
    http_response_code(401);
    exit('Invalid signature');
}

$event = json_decode(file_get_contents('php://input'));

// Handle different event types
switch ($event->type) {
    case 'quality.alert.created':
        sendCriticalAlert($event->data);
        break;
    
    case 'production.batch.completed':
        updateInventory($event->data);
        break;
    
    case 'forecast.updated':
        adjustProductionSchedule($event->data);
        break;
}

http_response_code(200);
?>
```

## Integration Best Practices

<Warning>
  **Security Best Practices**:
  
  - Never commit API keys to version control
  - Use environment variables for credentials
  - Rotate API keys quarterly
  - Use read-only keys for reporting
  - Enable IP whitelisting for production
  - Verify webhook signatures
</Warning>

<Note>
  **Performance Tips**:
  
  - Use webhooks instead of polling
  - Implement exponential backoff for retries
  - Cache frequently accessed data
  - Use GraphQL for complex queries
  - Batch operations when possible
  - Monitor rate limits
</Note>

## Troubleshooting

<Accordion title="Integration Not Syncing">
  **Check**:
  1. API credentials are valid and not expired
  2. IP address is whitelisted (if enabled)
  3. Integration status in dashboard shows "Connected"
  4. No recent error logs in Integration > Logs
  
  **Resolution**:
  - Disconnect and reconnect the integration
  - Verify permissions and scopes
  - Check third-party system status
  - Contact support if issue persists
</Accordion>

<Accordion title="Webhook Not Receiving Events">
  **Verify**:
  - Webhook URL is publicly accessible
  - SSL certificate is valid
  - Endpoint returns 200 status code
  - Signature verification is correct
  
  **Test**: Use webhook testing tool in dashboard to send test events
</Accordion>

<Accordion title="Data Discrepancies">
  **Common Causes**:
  - Time zone differences
  - Data mapping errors
  - Sync delays
  - Filtering differences
  
  **Resolution**: Check sync logs and verify field mappings in integration settings
</Accordion>
