### Querying Sensor Data

<CodeGroup>
```python Python SDK
from salsa_sdk import SalsaClient

client = SalsaClient(api_key="your_api_key")

# Get real-time sensor readings
sensors = client.iot.get_sensors(location="kitchen_1")

for sensor in sensors:
    reading = sensor.get_latest_reading()
    print(f"{sensor.name}: {reading.value} {reading.unit}")
    
    if reading.is_anomaly():
        print(f"⚠️  Alert: {reading.anomaly_details}")
        print(f"   Severity: {reading.severity}")
        print(f"   Recommended action: {reading.recommendation}")

# Query historical data
history = client.iot.query_history(
    sensor_id="temp_sensor_001",
    start_date="2024-01-01",
    end_date="2024-01-31",
    aggregation="hourly"
)

print(f"Average: {history.stats.mean}")
print(f"Min: {history.stats.min}")
print(f"Max: {history.stats.max}")
print(f"Anomalies detected: {history.stats.anomaly_count}")
```

```javascript JavaScript SDK
const SalsaClient = require('@salsa/sdk');
const salsa = new SalsaClient({ apiKey: 'your_api_key' });

// Real-time sensor monitoring
const sensors = await salsa.iot.getSensors({
  location: 'kitchen_1',
  types: ['temperature', 'ph', 'viscosity']
});

sensors.forEach(async (sensor) => {
  const reading = await sensor.getLatestReading();
  console.log(`${sensor.name}: ${reading.value} ${reading.unit}`);
  
  if (reading.isAnomaly) {
    console.log(`⚠️  ${reading.anomalyDetails}`);
  }
});

// Subscribe to real-time updates
const stream = salsa.iot.streamSensorData({
  sensor_ids: ['temp_001', 'ph_002'],
  interval: '1s'
});

stream.on('data', (reading) => {
  console.log(`${reading.sensor_id}: ${reading.value}`);
});
```

<Steps>
  <Step title="Plan Sensor Placement">
    Identify critical monitoring points:
    
    **Required Sensors**:
    - [ ] Refrigeration temperature sensors
    - [ ] Production area ambient sensors
    - [ ] At least one pH meter per production line
    
    **Recommended Sensors**:
    - [ ] Viscosity sensors for consistency
    - [ ] Color sensors for quality
    - [ ] Humidity sensors in storage
    - [ ] Equipment vibration monitors
    
    **Optional Sensors**:
    - [ ] Air quality monitors
    - [ ] RFID inventory tracking
    - [ ] Flow meters for precise measurement
  </Step>
  
  <Step title="Install Hardware">
    Install sensors according to manufacturer specifications:
    
    - Mount temperature sensors away from doors/vents
    - Install pH probes in easy-to-clean locations
    - Position vibration sensors on equipment housings
    - Ensure wireless sensors have strong signal strength
    
    <Note>
      **Professional Installation**: Enterprise customers receive complimentary professional installation and calibration.
    </Note>
  </Step>
  
  <Step title="Connect to Platform">
    Configure sensors in your Salsa dashboard:
    
```bash
# Add sensor via CLI
salsa iot add-sensor \
  --type temperature \
  --name "Walk-in Cooler" \
  --location kitchen_1 \
  --mac-address AA:BB:CC:DD:EE:FF \
  --alert-min 35 \
  --alert-max 40
```
    
    Or use the web dashboard: **Settings > IoT Devices > Add Sensor**
  </Step>
  
  <Step title="Calibrate & Test">
    Run calibration routines for accuracy:
    
    - pH meters: Calibrate with 4.0 and 7.0 buffer solutions
    - Temperature sensors: Verify against certified thermometer
    - Viscosity sensors: Test with known reference fluids
    - Color sensors: Calibrate with color reference cards
  </Step>
  
  <Step title="Configure Alerts">
    Set up alert thresholds and notification preferences:
    
    - Define critical, warning, and info alert levels
    - Configure SMS, email, and push notifications
    - Set escalation rules for unacknowledged alerts
    - Test alert delivery to all recipients
  </Step>
</Steps>

## Troubleshooting

<Accordion title="Sensor Offline or Not Reporting">
  **Check**:
  1. Battery level (wireless sensors)
  2. WiFi/Bluetooth signal strength
  3. Network connectivity
  4. Sensor placement (interference from metal/concrete)
  
  **Resolution**:
  - Replace batteries if < 20%
  - Move closer to gateway/router
  - Restart sensor by power cycling
  - Check gateway status in dashboard
</Accordion>

<Accordion title="Inaccurate Readings">
  **Common Causes**:
  - Sensor needs recalibration
  - Contamination on sensor probe
  - Sensor drift over time
  - Environmental interference
  
  **Resolution**:
  - Run calibration routine
  - Clean sensor probe per manufacturer specs
  - Replace sensor if > 2 years old
  - Check for nearby heat/cold sources
</Accordion>

<Accordion title="Too Many False Alerts">
  **Adjust Alert Thresholds**:
  
  Navigate to **Settings > IoT Devices > [Sensor Name] > Alert Settings**
  
  - Increase tolerance ranges slightly
  - Enable alert delay (e.g., alert only if out of range for 5+ minutes)
  - Use AI-recommended thresholds based on historical data
  - Set "quiet hours" for non-critical alerts
</Accordion>
