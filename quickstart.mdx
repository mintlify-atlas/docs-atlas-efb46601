## Welcome to Salsa as a Service v2.0

This quickstart guide will take you from zero to production in 15 minutes. You'll create an account, configure AI-powered forecasting, connect your POS system, and run your first smart production batch with real-time quality monitoring.

<Note>
  **What You'll Accomplish**: By the end of this guide, you'll have a fully configured account with AI forecasting enabled, your POS system integrated, and your first production batch tracked with quality metrics.
</Note>

## What You'll Need

Before getting started, gather these items:

- **Business email address** for account creation
- **POS system credentials** (Square, Toast, Clover, etc.) for integration
- **Historical sales data** (optional but recommended - 90+ days for better AI accuracy)
- **15 minutes** of your time

<Tip>
  **Pro Tip**: Having 90+ days of historical sales data will help the AI achieve 70-80% forecast accuracy from day one. Without historical data, the AI will still work but will need a few weeks to learn your patterns.
</Tip>

## Step 2: Configure AI Demand Forecasting

Enable AI-powered forecasting to predict salsa demand and optimize production:

<Steps>
  <Step title="Enable Forecasting">
    Navigate to **AI Features > Demand Forecasting** in your dashboard and toggle the feature on.

    Choose your forecast horizon:
    - **7 days** - Daily production planning (recommended for getting started)
    - **14 days** - Ingredient ordering and staffing
    - **30 days** - Strategic planning and seasonal preparation
  </Step>

  <Step title="Connect Data Sources">
    Link your systems to improve forecast accuracy:

    <CodeGroup>
      ```javascript JavaScript SDK
      import { SalsaClient } from '@salsaasaservice/sdk';

      const client = new SalsaClient({ 
        apiKey: 'your_api_key' 
      });

      // Configure forecast data sources
      await client.ai.configureForecast({
        dataSources: {
          posSales: true,              // Primary sales data
          weatherApi: 'openweathermap', // Weather impact
          events: ['eventbrite'],       // Local events
          reservations: true            // If applicable
        },
        modelParameters: {
          confidenceLevel: 0.95,
          seasonality: 'auto_detect',
          anomalyDetection: true
        }
      });
      ```

      ```python Python SDK
      from salsa_sdk import SalsaClient

      client = SalsaClient(api_key='your_api_key')

      # Configure forecast data sources
      forecast_config = client.ai.configure_forecast(
          data_sources={
              'pos_sales': True,
              'weather_api': 'openweathermap',
              'events': ['eventbrite'],
              'reservations': True
          },
          model_parameters={
              'confidence_level': 0.95,
              'seasonality': 'auto_detect',
              'anomaly_detection': True
          }
      )

      print(f"Forecast configured: {forecast_config.status}")
      ```

      ```bash cURL
      curl -X POST https://api.salsaasaservice.com/v2/ai/forecast/config \
        -H "Authorization: Bearer YOUR_API_KEY" \
        -H "Content-Type: application/json" \
        -d '{
          "data_sources": {
            "pos_sales": true,
            "weather_api": "openweathermap",
            "events": ["eventbrite"],
            "reservations": true
          },
          "model_parameters": {
            "confidence_level": 0.95,
            "seasonality": "auto_detect",
            "anomaly_detection": true
          }
        }'
      ```
    </CodeGroup>
  </Step>

  <Step title="Import Historical Data (Optional)">

## Step 3: Connect Your POS System

Integrate with your POS for real-time sales tracking and automated inventory sync:

<Tabs>

    **One-Click OAuth Setup** (Recommended)

    <Steps>
      <Step title="Navigate to Integrations">
        Go to **Integrations > POS Systems** and click **Connect Square**
      </Step>
      <Step title="Authorize Access">
        You'll be redirected to Square. Log in and click "Allow" to authorize Salsa as a Service to access:
        - Sales data
        - Inventory levels
        - Menu items
        - Location information
      </Step>
      <Step title="AI Menu Mapping">
        The AI automatically maps Square menu items to salsa inventory with 98% accuracy:
        
        **Detected Mappings:**
        - "House Salsa (Side)" → House Salsa Recipe → -4 oz per sale
        - "Burrito with Salsa" → House Salsa Recipe → -2 oz per sale

        Review and confirm mappings. Edit if needed.
      </Step>

        Toggle on "Real-Time Inventory Updates"
        
        Sales data syncs within 5 seconds. Every sale automatically:
        - Updates salsa inventory levels
        - Feeds into AI forecast model
        - Triggers reorder alerts if stock is low
      </Step>

## Troubleshooting Common Issues

<AccordionGroup>
  <Accordion title="AI Forecast seems inaccurate in first week">
    **Why**: The AI needs time to learn your specific patterns. Initial forecasts use industry averages.

    **Solution**: 
    - Import 90+ days of historical data for immediate accuracy
    - Continue normal operations; AI improves weekly
    - Typical timeline: 70% accuracy week 1 → 85% by week 4 → 95% by month 3
  </Accordion>

  <Accordion title="POS integration not syncing sales data">
    **Check these items:**
    1. POS credentials are correct and not expired
    2. API permissions include "read orders" and "read inventory"
    3. Menu mapping is complete (unmapped items won't sync)
    4. Network firewall allows outbound HTTPS to api.salsaasaservice.com

    **Test connection**: Go to **Integrations > POS > Test Connection**

    **Still not working?** Contact support with your integration logs from **Settings > Logs**
  </Accordion>

  <Accordion title="IoT sensors showing offline status">
    **Quick fixes:**
    1. Check sensor battery level (red LED = low battery)
    2. Verify IoT gateway is powered on (solid green light)
    3. Ensure gateway has WiFi connection
    4. Move sensors closer to gateway (max range: 100 feet)

    **Reset sensor**: Press and hold reset button for 5 seconds, then re-pair

    **Replace battery**: Use CR2032 batteries, available at any hardware store
  </Accordion>

  <Accordion title="Quality score lower than expected">
    **Common causes:**
    - **Temperature violations**: Check that ingredients stayed in range during prep
    - **Ingredient freshness**: Older ingredients reduce quality score
    - **Yield variance**: Large deviation from target affects consistency score

    **Review details**: Click on the batch to see quality breakdown and specific issues

    **Improve scores**: 
    - Maintain strict temperature control
    - Use ingredients within 2-3 days of receipt
    - Follow recipe quantities precisely
  </Accordion>

  <Accordion title="Can't access dashboard or API returns 401 errors">
    **Authentication issues:**
    1. Check that API key hasn't expired (**Settings > Developer > API Keys**)
    2. Verify you're using correct base URL: `https://api.salsaasaservice.com/v2`
    3. Ensure API key is in Authorization header: `Bearer YOUR_API_KEY`
    4. For dashboard access, clear browser cache and cookies, then log in again

    **Rate limits**: If you see 429 errors, you've hit rate limits. Upgrade plan or wait for reset.
  </Accordion>
</AccordionGroup>
